## NodeJS 정의

> NodeJS(이하 노드)는 크롬 V8 엔진으로 빌드된 자바스크립트 `런타임`이다.

즉, 자바스크립트로 작성된 **코드를 실행하는 환경**이다.  
웹 브라우저에서 자바스크립트 코드를 실행될 수 있는 것은 이 런타임을 내장하고 있기 때문이다.

노드를 사용하면 **브라우저 환경 밖**에서도 다양한 용도로 자바스크립트 코드를 실행할 수 있다.  
노드로 서버 역할을 하는 코드를 실행하면 서버 환경을 구축할 수 있다(노드가 서버는 아니다).

노드는 V8 엔진과 libuv 라이브러리를 사용한다.  
libuv 라이브러리는 노드의 특성인 이벤트 기반, 논 블로킹 I/O 방식을 구현하고 있다.

## NodeJS 특성

### 이벤트 기반(event-driven)

특정 이벤트(예. 클릭, 네트워크 요청, URL 입력 등)에 따라 다음 작업을 처리하는 방식이다.  
이벤트의 발생 시점이나 순서를 사전에 알 수 없기 때문에 이벤트 발생 시 수행할 작업을 미리 지정해놓아야 한다.

- `이벤트 리스너(event listener)`  
  이벤트를 감지(listen)하여 사전 등록된 콜백 함수를 호출하여 해당 이벤트를 처리하는 함수

### 논 블로킹 I/O

I/O 작업(예. http, DB CRUD, 파일 시스템 접근 등)을 수행할 때 이전 작업의 완료를 대기하지 않고 다음 작업을 처리하는 방식이다.  
노드가 동시 처리할 수 있는 작업을 모두 백그라운드로 넘기기 때문에 가능한 작업 방식이다.

- `백그라운드(background)`  
  OS 커널 혹은 libuv의 스레드 풀(thread pool)을 의미한다.  
  이벤트 리스너가 대기하는 곳으로 여러 작업이 동시 실행될 수 있다.
